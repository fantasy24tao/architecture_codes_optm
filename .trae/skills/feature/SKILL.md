# Feature Implementation Skill

## 概述

本技能用于指导开发人员安全地实现新功能或优化现有代码，确保代码质量和项目稳定性。

## 核心流程

### 1. 加载上下文 (Context Loading)

**目标**：深入理解项目架构和模块间的功能关系

**关键操作**：
- 分析项目的整体架构设计
- 理解核心模块之间的依赖关系
- 识别关键的数据流向和业务逻辑

### 2. 定位与规划 (Locate & Plan)

**目标**：找到相关代码并制定清晰的修改计划

**关键操作**：
- 使用 `ls`、`grep` 或 `read` 命令定位相关代码文件
- 深入理解现有逻辑，避免盲目修改
- 制定详细的修改计划并与用户沟通确认
- **重要**：切勿直接覆盖文件！

### 3. 代码实现 (Implementation)

**目标**：高质量地实现功能需求

**关键操作**：
- 保持与现有项目的**代码风格一致**（如命名规范、异步处理方式等）
- 代码应简洁高效，避免引入不必要的复杂性
- 非必要情况下，不修改无关代码
- 确保代码符合生产级标准

### 4. 自我验证 (Verification)

**目标**：确保修改的正确性和安全性

**关键操作**：
- 检查修改是否破坏了配置文件中的配置结构
- 确保没有引入文档中明确禁止的模式
- 验证代码逻辑的正确性
- 检查是否存在潜在的性能问题或安全漏洞

## 最佳实践提示

- 编写**生产级**代码，而非简单的 Demo
- 保持代码的可维护性和可扩展性
- 遵循团队的代码审查规范
- 及时更新相关文档
